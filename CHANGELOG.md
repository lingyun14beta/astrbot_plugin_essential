# Changelog

## v1.2.0

### 新增
- 戳一戳：被戳时随机回复一句话
- `/清除缓存`（仅管理员）：清除喜报、悲报、moe 的本地图片缓存
- mcs 服务器查询支持卡片图片渲染（需配置 t2i 服务）
- 所有指令加入 `desc` 说明，在面板指令管理页可见

### 修复
- `moe`：所有图片源均失败时不再崩溃；非 200 响应现在会继续尝试下一个源
- `今天吃什么`：食物列表为空时给出友好提示而非崩溃
- `喜加一`：促销列表为空时不再崩溃；解析单个游戏失败时跳过而非终止整个命令
- `搜番`：请求异常后正确清理等待状态，不再永久锁定用户；重复发命令时不再重置 30 秒计时
- `mcs`：MOTD 与"玩家人数"之间缺少换行的问题
- `早安/晚安`：重启后数据丢失的问题（写入时缺少外层 key）；睡眠时长异常值（超过 24 小时不再显示）；跨月同日期睡觉人数统计误判；隔离会话模式下每人都是"第一个睡觉的"问题

### 变更
- 早晚安数据迁移至 KV 存储，按群组拆分存储，不再依赖 `data/` 目录 JSON 文件
- 图片缓存目录迁移至 `data/plugin_data/astrbot_plugin_essential/`，不再写入工作目录
- 喜报、悲报、moe 图片改用 `{type}_{sender_id}.jpg` 命名，同用户重复调用覆盖而非无限累积
- `今天吃什么` 食物数据仍存储于 `resources/food.json`（与旧版一致）
- mcs HTML 模板重新设计，支持服务器图标、玩家进度条、渲染质量提升至 2x
- MOTD 中的 Minecraft 颜色代码（`§`）在渲染前自动过滤
- 命令解析改用切片而非 `replace`，避免内容中包含命令名时被误替换

### 需求
- 最低 AstrBot 版本：`>= 4.9.2`

### 升级注意
- 早晚安历史数据首次启动时自动迁移，无需手动操作
- 若旧版**未开启**隔离会话，历史睡眠记录可完整迁移
- 若旧版**已开启**隔离会话，历史睡眠记录无法迁移（key 格式不兼容），升级后历史数据丢失，之后的数据正常记录
